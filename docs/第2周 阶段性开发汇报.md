## 三、核心功能详解

### 3.1 Explore 模式（全新架构）

**技术方案**：原生 `aria_snapshot` 替代传统截图

**架构设计**：
```
用户输入 → Explore Agent → 页面快照（aria_snapshot）→ 动作解析 → 执行
                                    ↓
                            LoginGuard 检测
                                    ↓
                         搜索引擎拦截/跳过
```

**核心特性**：

| 特性 | 说明 | 状态 |
|------|------|------|
| aria_snapshot | 原生无障碍树快照，比截图更精准 | ✅ 完成 |
| LoginGuard | 多语言登录页面检测（中英文） | ✅ 完成 |
| 搜索引擎拦截 | 已在目标站点时跳过搜索脚本 | ✅ 完成 |
| Bing 搜索兜底 | 未知网站自动使用 Bing 搜索入口 | ✅ 完成 |
| 深度扫描 | 多步操作支持 | ✅ 完成 |
| 思考模式 | 支持 MiMo reasoning model | ✅ 完成 |
| 动作类型扩展 | 统一用户交互体验 | ✅ 完成 |
| Agent 重构 | 独立 Explore Agent 架构 | ✅ 完成 |

**MiMo 模型适配**：
- 修复 reasoning model 响应格式（`reasoning_content` 字段处理）
- 禁用 thinking mode 的 `chat_template_kwargs` 参数传递
- 使用 `response_format=json_object` 强制 JSON 输出
- 从 reasoning text 中提取 JSON 的容错机制

## 五、AI 能力增强

### 5.1 AI 辅助参数确认

**功能说明**：浏览器操作前自动确认技能参数，减少误操作

**实现方式**：
- 调用前通过 Panel 向用户确认参数
- 支持修改和取消操作

### 5.2 Strict Trigger AI 路由

**功能说明**：优化 AI 调用时机，减少无效调用

**路由逻辑**：
```
用户输入
    ↓
关键词快筛（规则匹配）
    ↓ [未命中]
LLM 精排（AI 解析意图）
    ↓ [未命中]
经验库查找
    ↓ [未命中]
生成临时脚本 → 沙箱执行 → 保存经验
```

### 5.3 MiMo 模型适配

**问题**：MiMo 是 reasoning model，响应格式与普通模型不同

**解决方案**：
- 识别 `reasoning_content` 字段
- 禁用 thinking mode 的参数传递
- 从 reasoning text 中提取 JSON
- 使用 `response_format=json_object` 强制输出
