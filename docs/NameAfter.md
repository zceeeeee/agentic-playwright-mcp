我建议不要直接沿用 **Claw / OpenClaw** 这条动物爪子路线，容易显得像跟随者。你们更突出的差异点是：**少调用视觉模型、少 token、用脚本/技能复用来完成桌面任务**。你给的 repo 也强调“让 AI Agent 写 Python 脚本来控制浏览器”，并且有技能库、经验库、自愈、视觉 fallback 这一套机制；README 里还写到复杂任务目前能力有限，说明名字最好别太“终局形态”，要能容纳早期阶段。([GitHub](https://github.com/zceeeeee/agentic-playwright-mcp))

## 带一点 “OpenClaw 竞品感” 的名字

OpenClaw 官方定位是能在你常用的聊天入口里“实际做事”的个人 AI 助手，它的 GitHub 描述也强调运行在自己的设备上、通过已有渠道交互。([OpenClaw](https://openclaw.ai/?utm_source=chatgpt.com)) 如果你们想让人一眼看出“类似但更轻量”，可以选这些：

| 名字            | 说明                         |
| --------------- | ---------------------------- |
| **FeatherClaw** | 有 Claw 的行动感，但强调更轻 |
| **LiteClaw**    | 直白：轻量版 OpenClaw 感     |
| **SoftClaw**    | 更温和、更省资源             |
| **SwiftPaw**    | 避开 Claw，但保留动物执行感  |
| **TinyClaw**    | 适合早期项目，但略显玩具感   |
| **LeanClaw**    | 技术感强，强调低成本         |

不过我不太建议最终用 **LiteClaw / FeatherClaw**，因为会把你们绑在 OpenClaw 的语义里。更好的方式是用自己的主线：**Feather / Script / Sparse / Desk / Pilot**。

## 我的最终排序

1. **FeatherDesk** — 最均衡，轻量、桌面、品牌感都好。
2. **ScriptPilot** — 最贴合技术内核，适合开发者。
3. **Glider** — 最像一个能长期发展的产品名。
4. **SparsePilot** — 最能表达少视觉调用、少 token。
5. **DeskSparrow** — 最有亲和力，适合早期桌面助手。
6. **SkillLoop** — 最适合开源框架/基础设施。
